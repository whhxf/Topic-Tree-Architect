# 双市场配置差异

## 市场判定规则

在 S1 阶段，当用户回答"目标市场"维度时，根据以下规则判定：

- **国内市场**：用户提及"中国大陆"、"百度"、"公众号"、"知乎"、"小红书"、"微信"、"企微"，或表示内容用中文发布
- **海外市场**：用户提及"Google"、"LinkedIn"、"英文"、"美国"、"欧洲"、"出海"等，或不在上述国内标识词范围内

判定结果存入 `ProductContext.target_market`：`"zh-CN"` 或 `"en-{区域代码}"`。

## 数据源配置

### 海外市场

| 需求 | 工具 | 获取数据 |
|---|---|---|
| 搜索量 + KD | Keywords Everywhere API | 月搜索量、竞争难度、CPC |
| 长尾词发现 | Tavily Search API | 真实搜索结果 + 相关问题 |
| SERP 竞品分析 | Tavily Search API | 排名前 10 页面标题/摘要 |

### 国内市场

| 需求 | MVP 方案 | Phase 2 升级 |
|---|---|---|
| 长尾词发现 | Tavily（目标站点设为 baidu.com） | 5118 API |
| 关键词热度 | LLM 知识库估算 | 百度指数 API（需合作资质） |
| 竞品内容分析 | Tavily（搜索百度结果） | 爱站 API |

**国内市场 MVP 注意**：搜索量数据缺失，必须在话题树节点上标注 `"data_confidence": "low"` 并向用户说明："搜索量数据参考价值有限，建议结合百度指数人工核验"。

## S1 阶段差异化追问

识别到目标市场后，切换以下追问角度：

| 维度 | 海外追问 | 国内追问 |
|---|---|---|
| 内容渠道 | "主要通过独立博客还是 LinkedIn/Medium？" | "内容主要发在官网博客、公众号，还是知乎/小红书？哪个渠道转化最好？" |
| 已有资产 | "独立站目前有多少篇博客文章？" | "官网博客、公众号历史文章各约多少篇？是否有知乎专栏？" |
| 竞品分析 | "竞品在 Google 上靠哪类内容获得流量？" | "竞品在百度/知乎/公众号上靠哪类内容获得曝光？" |
| 转化路径 | "目前用 Email 表单还是直接注册获取 Leads？" | "目前通过公众号引流、企微加好友，还是直接落地页转化？" |

## 话题树结构差异

### 海外：独立站为中心的单树

```
官网博客（唯一权威根基）
└── Pillar Page（3000+ 字终极指南）
    └── Cluster/Leaf（1500-2500 字深度文章）
内链网络绑定所有内容 → Google 视为完整知识体系
```

### 国内：官网为核心的多平台森林

```
官网博客（百度 SEO 根基，原创首发）
└── Pillar Page（2000+ 字，含清晰目录，注重收录速度）
    └── Cluster/Leaf（1000-1500 字，注重用户停留时长）

知乎专栏（品牌权威背书 + 外链来源）
└── 同一话题的"问答版"（不直接复制官网，差异化改写）
    └── 末尾植入官网链接（高权重外链）

微信公众号（私域流量主阵地）
└── 官网文章的精华提炼版（800-1000 字）
    └── 文末引导关注/加企微

小红书/抖音/B站（TOFU 种草）
└── 树叶内容切片为短视频/笔记，引流至官网或公众号
```

**实现要求**：国内市场时，`generate_topic_tree_json` 为每片树叶输出 `platform_versions` 字段（详见 tools-spec.md）。

## CTA 策略配置

### 海外市场

```
树干 CTA：Lead Magnet（PDF 白皮书/Checklist）→ 用 Email 换取下载 → 进入 Drip Campaign
树叶 CTA：免费试用按钮 / 预约 Demo 表单（BOFU 文章直接挂注册链接）
转化漏斗：邮件订阅 → 邮件序列培育（Mailchimp/ConvertKit）→ Demo → 付费
```

`cta_strategy` 字段示例：
```json
{
  "cta_type": "lead-magnet",
  "cta_copy": "Download the free VAT compliance checklist →",
  "cta_placement": "文末 + 文中自然断点"
}
```

### 国内市场

```
树干 CTA：扫码关注公众号 / 加企微，领取资料包 → 进入企微私域
树叶 CTA：文末引导加企微或扫码进群 → 运营人员 1v1 跟进
转化漏斗：公众号/企微 → 社群运营/个人号 → 内容培育 → 付费
```

`cta_strategy` 字段示例：
```json
{
  "cta_type": "enterprise-wechat",
  "cta_copy": "扫码加企微，免费领取《行业内容营销实战手册》",
  "cta_placement": "文末 + 文章三分之二处",
  "wechat_ecosystem": {
    "official_account": true,
    "enterprise_wechat": true
  }
}
```

## SEO 核心动作差异

| 动作 | 海外（Google） | 国内（百度） |
|---|---|---|
| 内链 | **绝对核心**——Pillar-Cluster 依赖内链成立 | 有效但依赖度低于 Google，与外链并重 |
| 外链 | 中等权重，低质量外链有惩罚风险 | **高权重**，来自知乎/36kr/虎嗅的外链显著拉升排名 |
| 原创度 | 通过 E-E-A-T 信号体现（作者 Bio、独家数据） | 直接信号——百度有原创保护，**首发优势明显** |
| 收录速度 | Google Search Console 提交 Sitemap | 百度搜索资源平台主动推送 URL（IndexNow），更敏感 |
| 用户行为 | 影响较小 | **显著影响排名**——停留时长、跳出率权重高 |

**S4 国内市场额外输出**：
- 外链获取建议（推荐投稿：知乎、36kr、少数派、虎嗅、掘金）
- 百度收录推送提醒（每次发布后主动提交 URL）

## 效果追踪配置

| 工具 | 海外市场 | 国内市场 |
|---|---|---|
| 排名监控 | Google Search Console | 百度搜索资源平台 |
| 流量分析 | Google Analytics 4 | 百度统计 |
| 施肥动作 | 增加内容深度 + 内链权重 | 增加内容深度 + 更新时间戳 + 主动推送 URL |
| 剪枝动作 | 合并 + 301 重定向 | 合并 + 百度平台提交死链通知 |
